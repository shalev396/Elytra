# Elytra Templates

This directory contains templates for various project components that can be generated by the Elytra CLI.

## Directory Structure

- `frontend/`: Frontend framework templates (React, Vue.js, Angular, Svelte, Next.js, Nuxt.js)
- `backend/`: Backend framework templates (Express.js, NestJS, Fastify, Serverless Functions)
- `database/`: Database configuration templates (PostgreSQL, MySQL, MongoDB, SQLite)
- `auth/`: Authentication templates (Email/Password, OAuth 2.0, JWT, Auth0, Firebase Authentication)
- `features/`: Additional feature templates (RBAC, Pricing, API Documentation, etc.)

## Creating Templates

Templates use a simple placeholder system with the format `{{placeholderName}}`. These placeholders are replaced with actual values during project generation.

Common placeholders include:

- `{{projectName}}`: The name of the project
- `{{projectDescription}}`: The description of the project
- `{{packageManager}}`: The package manager (npm, yarn, pnpm)
- `{{database}}`: The selected database
- `{{authentication}}`: The selected authentication method

## Template Structure

Each template should follow a consistent structure:

- Include a `template.json` file with metadata about the template
- Organize files in a way that mirrors their final structure in the generated project
- Include a README.md with instructions specific to the template

## Example Template

A simple React template might look like:

```
frontend/
└── react/
    ├── template.json
    ├── package.json
    ├── src/
    │   ├── App.tsx
    │   ├── index.tsx
    │   └── components/
    │       └── Example.tsx
    └── README.md
```

With a `template.json` like:

```json
{
  "name": "React",
  "description": "A basic React template with TypeScript",
  "version": "1.0.0",
  "dependencies": {
    "react": "^18.2.0",
    "react-dom": "^18.2.0"
  }
}
```
